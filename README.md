<h1>Домашнее задание 1</h1>

<p>
  Вам нужно реализовать CLI приложения. Кроме кода, вам необходимо в качестве артефактов
  приложить текстовый файл с тем, как вы проверяли работоспособность вашего кода
  (просто копия команд и выводов из терминала).
</p>

<h2>1.1</h2>

<p>
  Написать упрощённый вариант утилиты <code>nl</code> — скрипт, который выдаёт в
  <code>stdout</code> пронумерованные строки из файла.
</p>

<p>
  Если файл не передан, то скрипт читает строки из <code>stdin</code>.
</p>

<p>
  Он должен работать так же, как:
</p>

<pre><code>nl -b a
</code></pre>

<h2>1.2</h2>

<p>
  Написать упрощённый вариант утилиты <code>tail</code> — скрипт, выводящий в
  <code>stdout</code> последние 10 строк каждого из переданных файлов.
</p>

<ul>
  <li>
    если передано больше одного файла, то перед обработкой очередного файла необходимо
    вывести его имя. Подробности смотрите в оригинальной утилите <code>tail</code>,
    ваш скрипт должен повторять форматирование;
  </li>
  <li>
    если не передано ни одного файла, то нужно вывести последние 17 строк из
    <code>stdin</code>.
  </li>
</ul>

<h2>1.3</h2>

<p>
  Написать скрипт, работающий так же, как утилита <code>wc</code>, вызванная без
  дополнительных опций.
</p>

<p>
  То есть для каждого переданного файла утилита выводит статистику (3 числа) и имя файла.
</p>

<p>При этом:</p>

<ul>
  <li>
    если передано больше одного файла, то в самом конце утилита выводит суммарную
    статистику (<code>total</code>);
  </li>
  <li>
    если ни одного файла не передано, то утилита считывает весь вход и печатает
    для него статистику без имени.
  </li>
</ul>

<h1>Домашнее задание 2</h1>

<p>
  Необходимо реализовать генератор LaTeX в функциональном стиле.
  Для генерации латеха нельзя использовать сторонние библиотеки.
</p>

<h2>2.1</h2>

<p>
  Написать функцию для генерации таблиц (<code>.tex</code>).
  На вход поступает двойной список, на выходе строка с
  отформатированным валидным латехом.
</p>

<p>
  Проверить, что латех валидный можно, например, в
  <a href="https://www.overleaf.com/" target="_blank">Overleaf</a>.
</p>

<p>
  Используя другой модуль (<code>.py</code> файл) нужно импортировать вашу функцию
  и используя её необходимо сохранить в <code>.tex</code> файл пример работы этой
  функции, это и будет артефактом для этой задачи.
</p>

<p>
  Если вы никогда не работали с латехом, то это хороший повод попробовать.
</p>

<h2>2.2</h2>

<p>
  Написать функцию для генерации картинок в латех.
  (В том же модуле, что и предыдущую функцию)
</p>

<p>
  В качестве картинки использовать любую картинку (<code>png</code> файл), НО:
</p>

<ul>
  <li>
    Нужно собрать ваш код в библиотеку при помощи любого build
    frontend/backend, выложить в любой репозиторий
  </li>
  <li>
    Установить вашу библиотеку
  </li>
</ul>

<p>
  После этого, используя вашу библиотеку, сгенерировать по полученному латеху
  PDF с таблицей из задачи 2.1 и картинкой.
</p>

<p>
  <strong>PDF</strong> — первый артефакт задачи,
  <strong>ссылка на репозиторий в PyPI/Anaconda</strong> — второй.
</p>

<p>
  Генерировать pdf можно при помощи <code>pdflatex</code>.
  Но чтобы он заработал, нужен дистрибутив самого tex.
  Их много разных.
</p>

<h1>Домашнее задание 3</h1>

<p>
  Нужно реализовать небольшую библиотеку для работы с матрицами.
</p>

<h2>3.1</h2>

<p>
  Сделать класс матрицы, в котором определить операции сложения и умножения (матричного и покомпонентного) через перегрузку операторов +, *, @ (как в numpy). Вызывать исключения, если матрицы на входе некорректной      размерности (ValueError)
Сгенерировать две матрицы через <code>np.random.randint(0, 10, (10, 10))</code>code> c seed-ом 0 и над ними провести все три операции. Записать результаты в текстовые файлы, названные <code>matrix+.txt, matrix*.txt, matrix@.txt</code>code>, соответственно. Это будет артефактом задачи.
</p>

<p>
  <b>Комментарий студента</b> в Windows символ '*' запрещён в именах файлов,
поэтому имена файлов были изменены на:
matrix_mul.txt, matrix_plus.txt, matrix_matmul.txt
</p>

</p>

<p>
  Также добавить через примеси: запись объекта в файл, красивое отображение в консоли (__str__), getter-ы и setter-ы для полей класса
В самих классах должно быть минимальное количество методов
Артефакты задачи - аналогично задаче 3.1
